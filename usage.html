As SSH uses public-key cryptography to authenticate the remote computer, a private and public key pair must be generated to be able to authenticate the remote client. These keys are asymmetric. This negates the need for a passord and this is also known as passwordless SSH.

The private key is accesible solely by its owner and is used to authenticate the user. If this key was known to anyone else, it could be used to inpersonate the initial user. The corresponding public key can be given to any entity that wants to authenticate the user and it does not matter who has access to this key as it means nothing without its corresponding private key. To read more about public-key cryptography/ asymmetric cryptography go here.

Ahead I will be showing you how to generate SSH keys using the Ed25519 key type.

Generating SSH keys on Windows

Checking for existing SSH keys

Open the command prompt, and run:

cd %userprofile%/.ssh

If there is no such file of directory then no such SSH keys exist and you can go to "Generating an SSH key"

To check to see if you have a key already run:

dir id_*

If a key exists already you can use that one. If you would like to generate a new one you may want to back this one up by running:

mkdir key_backup

copy id_ed25519* key_backup

Generating an SSH key

ssh-keygen -t ed25519 -C "your_email@example.com"

Press enter to accept the default file location and follow the prompted steps. The following interaction will look like this:

C:\Users\ASUS>ssh-keygen -t ed25519 -C "your_email@example.com"

Generating public/private ed25519 key pair.

Enter file in which to save the key (/c/Users/ASUS/.ssh/id_ed25519):

Created directory '/c/Users/ASUS/.ssh'.

Enter passphrase (empty for no passphrase):

Enter same passphrase again:

Your identification has been saved in c/Users/ASUS/.ssh/id_ed25519.

Your public key has been saved in c/Users/ASUS/.ssh/id_ed25519.pub.

The key fingerprint is:

SHA256:wvaHYeLtY6+DlvV5sFZgDi3RTAe8R2KY23N78qrGqJE your_email@example.com

Generating SSH keys on Linux and macOS

Checking for existing SSH keys

Open the terminal, and run:

cd ~/.ssh

If there is no such file of directory then no such SSH keys exist and you can go to "Generating an SSH key"

To check to see if you have a key already run:

ls id_*

If a key exists already you can use that one. If you would like to generate a new one you may want to back this one up by running:

mkdir key_backup

cp id_ed25519* key_backup

Generating an SSH key

ssh-keygen -t ed25519 -C "your_email@example.com"

Press enter to accept the default file location and follow the prompted steps. The following interaction will look like this:

chaiss@C02FP1JWMD6V ~ % ssh-keygen -t ed25519 -C chaiss@email.com

Generating public/private ed25519 key pair.

Enter file in which to save the key (/Users/chaiss/.ssh/id_ed25519):

Enter passphrase (empty for no passphrase):

Enter same passphrase again:

Your identification has been saved in /Users/chaiss/.ssh/id_ed25519.

Your public key has been saved in /Users/chaiss/.ssh/id_ed25519.pub.

The key fingerprint is:

SHA256:gTVWKbn41z6JgBNu3wYjLC4H6oW5bmhaXyaVABwmEtE chaiss@email.com

The key's randomart image is:

|==+.    +o..     |
|.oE.   +o..      |
|    . ...o       |
|     .o...       |
|     oo+S  .     |
|  + ..B = . .    |
|.+.+.oo+ * o .   |
|o++.o+  . + +    |
|B+ o.    .   .   |
+----[SHA256]-----+

chaiss@C02FP1JWMD6V ~ %
